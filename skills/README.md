# OpenClaw Skills

TypeScript skills the agent loads, organized one folder per skill.

## Layout

```
skills/
  shared/              code every skill uses
    db.ts              pooled, read-only MySQL query() helper
  propertySearch/      Week 2–3: NL property search skill
    parse.ts           free-text query → PropertyFilter        (Week 2)
    search.ts          PropertyFilter → ListingRow[]            (Week 3)
    format.ts          ListingRow[] → formatted property cards  (Week 3)
    index.ts           propertySearchSkill: the composition (public entry point)
    cities.json        parser gazetteer (generated by generate_cities.py)
    *.test.ts          co-located tests
```

Convention: outsiders import a skill by its folder (`propertySearch/index.ts`);
the internal files (`parse`/`search`/`format`) import each other. Shared code
lives in `shared/`, never inside a single skill.

## Running the tests

From the repo root, install once then run:

```bash
npm install      # pulls tsx (TS runner) into node_modules
npm test         # runs the parser cases → "All 15 cases passed"
```

`npm test` runs the parser (`parse.test.ts`) against labeled query→filter cases
and exits non-zero if any fail — no DB needed.

The DB-backed smoke test needs your `.env` + MySQL running:

```bash
npx tsx skills/propertySearch/search.test.ts
```
